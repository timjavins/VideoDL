# Issue Tracker

This repo uses GitHub Issues as the source of truth for work tracking.

## How skills should work

- Use `gh issue create`, `gh issue view`, and related GitHub CLI commands.
- Treat issues in the repository GitHub project as canonical.
- Do not create `.scratch/` markdown issues unless the tracker is explicitly switched later.

## Practical notes

- Reference issues and PRs by GitHub number.
- Prefer linking to the GitHub issue rather than duplicating long task descriptions in docs.
