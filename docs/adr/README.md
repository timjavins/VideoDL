# ADRs

This directory stores architecture decision records for VideoDL.

## How to use

- Add a new ADR when a decision changes project architecture, data flow, API shape, or operational behavior.
- Keep ADRs short, dated, and focused on one decision.
- Update the root `CONTEXT.md` when a decision becomes part of the project vocabulary or operating rules.

## Suggested format

- Title
- Status
- Context
- Decision
- Consequences
- Notes
